## 跳跃问题
给定一个长度为 n 的 0 索引整数数组 nums。初始位置在下标 0。

每个元素 nums[i] 表示从索引 i 向后跳转的最大长度。换句话说，如果你在索引 i 处，你可以跳转到任意 (i + j) 处：

0 <= j <= nums[i] 且
i + j < n
返回到达 n - 1 的最小跳跃次数。测试用例保证可以到达 n - 1
#### 感悟
首先我需要知道每个节点我的跳跃潜力是多是.那么我再0位置,那么我的跳跃边界就是0加我的跳跃潜力;
我的跳跃潜力是固定的,但是也是由上一次跳跃点决定的,那么我只需要让我的潜力最大就好
为什么呢?我知道你有一个疑问,就是当我所处在最开始的节点a,我的跳跃范围是b,也就是a到b的任意一个地方我随意都能跳到
我为什么要去选下一个跳跃节点,选择那个处在ab之间的跳跃节点最强跳跃节点c呢?万一他的跳跃能力不行,导致我想跳d够不到怎么办?
你注意,因为c是节点是预知我下次跳跃最强的节点,所以c如果能够到d,的前提是什么?是首先a够不到d,所以要借助一个跳跃能力最远的去够到d
如果说哪怕是跳跃能力最强的c够不到d,那么a到b之间的点一定都够不到d,因为c就是最强的跳跃能力,他的范围最大最长.
那如果说我能找到一个节点处于a到b之间的m,m也能到d,那我为什么要选c?
废话当然也许会有别的点能够到d,我们要求的是最短次数,选择c一定是最小次数,因为选c,一定能跳到d,而选m,也许能跳到且也是一次机会,但是他不是最远边界,他有不确定 你能不能够得到
#### 代码实现
```java
class Solution {
    public int jump(int[] nums) {
        // 如果数组长度只有1，说明我一开始就在终点了，不用跳
        if (nums.length <= 1) return 0;

        int time = 0;          // 我的跳跃次数
        
        int maxPotential = 0;  // 这代表我所说的“最强跳跃节点c能到达的最远极限”
        int currentBound = 0;  // 这代表我所说的“a到b的范围”，也就是我当前这一步必须走完的边界b

        // 定义指针 po，我只需要走到倒数第二个节点去寻找潜力就行了
        for (int po = 0; po < nums.length - 1; po++) {
            
            // 1. 我在 a 到 b 的范围内走动，一边走一边寻找那个跳跃能力最强的节点 c
            // 它的潜力就是当前的 po 加上它对应的 nums[po]
            maxPotential = Math.max(po + nums[po], maxPotential);

            // 2. 我必须要有一个 sign，告诉我什么时候必须记一次跳跃（time++）
            // 当我的指针 po 走到了当前范围的边界 b（也就是 currentBound）时
            // 说明 a 到 b 我逛完了，我必须起跳了！
            if (po == currentBound) {
                time++; 
                
                // 起跳后，我下一次的边界，自然就是我刚才在 a 到 b 里找到的最强节点 c 的极限
                currentBound = maxPotential; 
                
                // 既然找到了极限，如果这个极限已经能摸到我想跳的终点 d 了
                // 那我就没必要往后走了，这就已经是最少次数了
                if (currentBound >= nums.length - 1) {
                    return time;
                }
            }
        }
        
        return time;
    }
}`
```

## 糖果分配问题
n个孩子站成一排。给你一个整数数组 ratings 表示每个孩子的评分。

你需要按照以下要求，给这些孩子分发糖果：

每个孩子至少分配到 1 个糖果。
相邻两个孩子中，评分更高的那个会获得更多的糖果。
请你给每个孩子分发糖果，计算并返回需要准备的 最少糖果数目
#### 感悟或者理解
首先孩子一共是n个,我最开始的初步解法就是,准备一共数组,也就是一共装每个孩子应该要多少个糖果的桶子,因为孩子最少肯定是一个糖果,所以每个孩子的筒子加一,然后我们开始进行坡度比较,每个孩子是依据评分,什么评分?相邻的两个或者一个孩子的评分来比较的
那么也会是说,我设立两个双指针,挨个遍历每个孩子,假如说左边有个孩子,那么他右边就是一个孩子啊,我让指针指向他们,然后就是能读取他们的信息,无论大小,总有加减,全部装到对应索引的桶里就是最好的

但是这样会造成一个问题?
举个极端的例子： 假设 ratings = [3, 2, 1]。
按照正确的逻辑，糖果应该是 [3, 2, 1]。
如果按照你的单向遍历逻辑去走（假设修复了下面提到的那些变量名 bug，且初始糖果都是 1）：

第一步：比较 l=0, r=1（评分 3 和 2）。3 > 2，所以把左边（3）的糖果加 1，变成 2。此时糖果数组是 [2, 1, 1]。

第二步：比较 l=1, r=2（评分 2 和 1）。2 > 1，所以把左边（2）的糖果加 1，变成 2。此时糖果数组是 [2, 2, 1]。
问题出现了！ 当第二步更新了中间孩子的糖果后，第一个孩子和第二个孩子的糖果变成一样多了（都是 2），但这违背了题目“评分 3 > 评分 2，糖果也必须更多”的规则。
结论： 只从左往右走一遍是不行的，因为右边发生的变化，无法“倒退”回去通知左边。这就是为什么经典的贪心解法必须左到右走一遍，右到左再走一遍


这样就需要我们反向在遍历一边,这样就是完美的,由于遍历两边加上要遍历一个同,时间复杂度就是log4n

#### 解法一
```java
import java.util.Arrays;

class Solution {
    public int candy(int[] ratings) {
        int n = ratings.length;
        int[] count = new int[n];
        
        // 1. 每人至少一个糖果
        Arrays.fill(count, 1);
        
        // 2. 从左往右遍历：只保证右边评分比左边高时，右边糖果比左边多
        for (int i = 1; i < n; i++) {
            if (ratings[i] > ratings[i - 1]) {
                count[i] = count[i - 1] + 1;
            }
        }
        
        // 3. 从右往左遍历：只保证左边评分比右边高时，左边糖果比右边多
        for (int i = n - 2; i >= 0; i--) {
            if (ratings[i] > ratings[i + 1]) {
                // 注意这里要取最大值：
                // 既要满足左到右的规则，也要满足右到左的规则
                count[i] = Math.max(count[i], count[i + 1] + 1);
            }
        }
        
        // 4. 统计总数并返回
        int sum = 0;
        for (int c : count) {
            sum += c;
        }
        
        return sum;
    }
}
```
